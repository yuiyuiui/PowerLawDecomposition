# Matrix Pencil 方法误差来源分析报告

## 0. 问题

对 10 项指数和 $f(x) = \sum_{n=1}^{10} e^{-0.5n \cdot x}$ 在网格 $[1, 5]$（h=0.1, M=41）上，
用 matrix pencil 方法恢复参数 $a_n = 0.5n$。Float64 下恢复精度很差（max|Δa| ~ 3.5），
但 BigFloat 下几乎完美（max|Δa| ~ 10⁻¹³⁶）。

三个可能的误差来源：
1. **输入信号的机器精度限制**（Float64 噪声地板 ~10⁻¹⁶）
2. **算法本身的设计**（矩阵铅笔法的参数选择）
3. **SVD 求解中的 Float64 精度不足**

## 1. SVD 奇异值谱分析

Hankel 矩阵 Y₀（21×20）的奇异值谱揭示了问题的本质：

| k | S[k] (Float64) | S[k] (BigFloat) | S[k-1]/S[k] 比值 |
|---|----------------|-----------------|------------------|
| 1 | 8.867e+00 | 8.867e+00 | — |
| 2 | 3.784e-01 | 3.784e-01 | 23 |
| 3 | 1.266e-02 | 1.266e-02 | 30 |
| 4 | 2.929e-04 | 2.929e-04 | 43 |
| 5 | 4.182e-06 | 4.182e-06 | 70 |
| 6 | 3.609e-08 | 3.609e-08 | 116 |
| 7 | 1.874e-10 | 1.874e-10 | 193 |
| 8 | 5.647e-13 | 5.646e-13 | 332 |
| **9** | **8.852e-16** | **8.958e-16** | 630 |
| **10** | **5.224e-16** | **5.694e-19** | Float64: **1.7** / BigFloat: **1573** |
| 11 | 4.881e-16 | 3.113e-77 | Float64: 1.1 / BigFloat: 1.8e+58 |

**关键发现**：
- S[1] 到 S[8] 在两种精度下完全一致（真实信号分量）
- S[9] ≈ 9e-16 在两种精度下也一致（但已接近 Float64 eps）
- **S[10]**: Float64 给出 5.22e-16 ≈ S[9]（无法区分信号与噪声），BigFloat 给出 5.69e-19（比 S[9] 小 1573 倍，清晰可分辨）
- Float64 的条件数 S[1]/S[10] = 1.7×10¹⁶ ≈ 1/eps(Float64)，**恰好在精度极限**

**结论**：Float64 的量化噪声将第 10 个奇异值 "抬高" 了 ~1000 倍（从 5.7e-19 到 5.2e-16），使其淹没在噪声地板中。

## 2. 误差源隔离实验

设计三组对照实验：

| 场景 | 输入数据 | 计算精度 | 测量的是什么 |
|------|---------|---------|------------|
| A | BigFloat (512bit) | BigFloat | 纯算法误差 |
| B | Float64 → BigFloat | BigFloat | 输入噪声传播 |
| C | Float64 | Float64 | 完整问题（三源叠加） |

### 结果

| 场景 | max\|Δa\| | 说明 |
|------|----------|------|
| **A** | **2.3×10⁻¹³⁶** | 算法本身几乎完美 |
| **B** | **1.4×10¹** | Float64 数据 + BigFloat 算数 → 仍然很差 |
| **C** | **7.6×10⁰** | Float64 全链路 → 与 B 相当 |

### 归因

| 误差源 | 量化指标 | 结论 |
|--------|---------|------|
| **Source 1（输入精度）** | B/A = **6.1×10¹³⁶** | **绝对主因** |
| Source 2（算法设计） | 见实验 6 | 次要因素，可通过调参改善 |
| **Source 3（SVD 精度）** | C/B = **0.53** | **不是问题** |

**场景 B 和 C 的逐分量对比**：

| 分量 | a_true | 场景 B error | 场景 C error |
|------|--------|-------------|-------------|
| 1 | 0.5 | 2.2e-07 | 3.8e-07 |
| 2 | 1.0 | 5.7e-05 | 1.0e-04 |
| 3 | 1.5 | 2.7e-03 | 5.4e-03 |
| 4 | 2.0 | 3.5e-02 | 7.6e-02 |
| 5 | 2.5 | 9.7e-02 | 1.8e-01 |
| 6 | 3.0 | 2.6e-02 | 8.1e-02 |
| 7 | 3.5 | 1.8e-01 | 1.4e-01 |
| 8 | 4.0 | 3.9e-01 | 3.8e-01 |
| 9 | 4.5 | 4.9e-01 | 4.9e-01 |
| 10 | 5.0 | **14.2** | **7.6** |

B 和 C 的误差量级完全一致（同一量级），证实：
- **Float64 SVD 的计算误差可以忽略**
- 误差完全由输入数据的 Float64 噪声决定

## 3. 网格参数的影响

### 3a. 域起点 L₀ 的影响

固定 h=0.1, 域长度=4, Float64:

| L₀ | max\|Δa\| | 改善 | 原因 |
|----|----------|------|------|
| 0.0 | 0.227 | 15× | 包含 x=0 处所有分量同时为 1 的信息 |
| 0.5 | 0.464 | 8× | |
| 1.0 | 3.49 | 基准 | |
| 2.0 | 3.17 | 1.1× | 慢衰减分量信息减少但更集中 |

**启示**：从 x=0 开始采样能将最大误差降低 15 倍，因为 $e^{-a_n \cdot 0} = 1$ 对所有 $n$ 成立，
给 Hankel 矩阵提供了所有分量的最大信号。

### 3b. 步长 h 的影响

固定 L₀=1, 域=[1,5], 默认 L=M/2:

| h | M | cond(Y₀) | max\|Δa\| | S[10]/eps |
|---|---|----------|----------|----------|
| 0.2 | 21 | 4.5e16 | 1.23 | 0.5 |
| 0.1 | 41 | 1.7e16 | 3.49 | 2.4 |
| 0.05 | 81 | 1.1e16 | 37.4 | 7.3 |
| 0.02 | 201 | 1.0e16 | 64.9 | 18.3 |
| 0.01 | 401 | 1.5e15 | 632 | 241 |

**反直觉结果**：更密的网格 → 更差的恢复！原因：
- 虽然 S[10]/eps 增大（信号更强），但默认 L = M/2 随 M 线性增长
- Hankel 矩阵尺寸从 21×20 增至 201×200，噪声子空间维度急剧增大
- **这是 Source 2（算法设计）的体现 — L 的默认选择不适合过采样场景**

### 3c. 域长度（数据量）的影响

固定 L₀=1, h=0.1:

| M | 域 | max\|Δa\| | 改善 |
|---|---|----------|------|
| 21 | [1,3] | 14.3 | — |
| 41 | [1,5] | 3.49 | 4× |
| 91 | [1,10] | 0.246 | 14× |
| 191 | [1,20] | **0.0103** | **337×** |

**最有效的改进**：延长采样域。从 [1,5] 到 [1,20] 带来 337 倍改善。

原因：更长的域让快衰减分量有更多空间展示其指数特征，
增加了 Hankel 矩阵中的有效信息量。

## 4. Pencil 参数 L 的优化

当 M >> 2N+1 时，默认 L = M/2 严重次优。

### 域 [1,5], h=0.1, M=41

| L | max\|Δa\| | 说明 |
|---|----------|------|
| **10 = N** | **0.572** | 最小 L，最佳噪声抑制 |
| 12 | 0.511 | |
| 15 | 3.84 | 开始劣化 |
| 20 = M/2 | 3.49 | 默认值，最差 |

### 域 [0,20], h=0.1, M=201

| L | max\|Δa\| | 说明 |
|---|----------|------|
| 15 | 0.202 | |
| 20 = 2N | 0.159 | |
| **30 = 3N** | **0.0029** | 最优区间 |
| 40 = 4N | 0.0022 | |
| 50 = 5N | **0.0011** | 接近饱和 |
| 100 = M/2 | 0.0011 | 未进一步改善 |

**规律**：
- 数据量不足（M ≈ 2N+1）时：L ≈ N 最优
- 数据充足（M >> 2N+1）时：L ≈ 3N~5N 最优
- L = M/2 在过采样时浪费了 SVD 的降噪能力

## 5. 各分量的信息论分析

每个分量 $e^{-a_n x}$ 在域 [1,5] 上的信号强度：

| n | aₙ | 信号范围 [x=1, x=5] | 信号/噪声比 (SNR) |
|---|-----|---------------------|-------------------|
| 1 | 0.5 | [0.607, 0.082] | 1.8×10¹⁵ |
| 2 | 1.0 | [0.368, 6.7e-3] | 1.1×10¹⁵ |
| 3 | 1.5 | [0.223, 5.5e-4] | 6.6×10¹⁴ |
| ... | ... | ... | ... |
| 8 | 4.0 | [0.018, 2.1e-9] | 5.4×10¹³ |
| 9 | 4.5 | [0.011, 1.7e-10] | 3.3×10¹³ |
| 10 | 5.0 | [0.0067, 1.4e-11] | 2.0×10¹³ |

虽然每个分量在 x=1 处的 SNR 都很高（~10¹³），但问题不在于**单点 SNR**，
而在于 Hankel 矩阵的**整体条件数**。

10 个分量的指数基函数构成的 Vandermonde 矩阵条件数 ~ 10¹⁶，
这意味着需要 ~16 位有效数字来完全分离所有分量 — 恰好是 Float64 的极限。

## 6. 综合结论

### 三个误差源的定量排序

| 排名 | 误差源 | 贡献度 | 证据 |
|------|--------|-------|------|
| **1** | **输入精度（Source 1）** | **~100%** | B/A = 10¹³⁶，输入噪声完全决定结果 |
| 2 | 算法设计（Source 2） | 可优化 | 调整 L 和域参数可改善 10~1000× |
| 3 | SVD 精度（Source 3） | **可忽略** | C/B = 0.53，Float64 SVD 足够 |

### 为什么 Source 1 是主因？

Matrix pencil 的核心是从 Hankel 矩阵的 SVD 中提取信号极点。
10 个指数分量产生 10 个非零奇异值，但在 Float64 下：

$$S_{10}^{\text{true}} = 5.69 \times 10^{-19} \ll \epsilon_{\text{Float64}} \times S_1 \approx 2 \times 10^{-15}$$

第 10 个（甚至第 9 个）奇异值**已经低于 Float64 噪声地板**。
算法无法区分真实信号和数值噪声，导致恢复失败。

这不是算法的缺陷，而是**信息论的硬限制**：
Float64 的 ~16 位有效数字无法承载条件数 ~10¹⁶ 的问题。

### 针对性改进方案

| 方案 | 改善 | 代价 | 适用场景 |
|------|------|------|---------|
| **全链路 BigFloat** | ~10¹³⁶× | 计算慢 10~100× | 需要精确恢复所有分量 |
| 延长采样域 | 10~300× | 需要更多数据 | 可控制采样范围 |
| 从 x≈0 开始 | 15× | 可能不实际 | 可以接触信号起始段 |
| 优化 pencil_L ≈ 2N~3N | 2~10× | 无代价 | 总是应该做 |
| 减少请求分量数 | 5~10× | 丢失弱分量 | 只需要主要分量 |
| 自适应 SVD 截断 | 2~5× | 无代价 | 总是应该做 |

### 改进后的推荐用法

```julia
# 改进前（默认参数）
res = matrix_pencil(grid, f, N)

# 改进后
res = matrix_pencil(grid, f, N;
    pencil_L = 3 * N,             # 控制 pencil 参数
    svd_tol = eps(eltype(f)) * 10  # 自适应截断
)
```

### 实际建议

1. **如果数据精度是瓶颈**（绝大多数实际场景）→ 优化数据采集（更长域、从 x≈0 开始），
   并用 `pencil_L ≈ 3N` 和 `svd_tol > 0`
2. **如果需要恢复所有分量到高精度** → 必须用 BigFloat 全链路（输入 + 计算）
3. **如果分量数 N 未知** → 先用大 N 运行，看 SVD 奇异值的 "断崖" 位置确定有效分量数
